# TOOLS.md — Голос 🎙️

## Файлы, с которыми работаю

- `BUSINESS.md` (в shared/ или корне воркспейса) — услуги, цены, график,
  адрес, мастера. **Единственный источник фактов о бизнесе.**
- `MEMORY.md` — моя память (клиенты, спящие, отзывы, дедуп).
- `~/.aistack/events.log` — событийная шина команды.

## Событийная шина — как писать

Одна строка JSON, append в конец файла:

```json
{"id":"<uuid4>","ts":"<ISO-время>","from":"voice","type":"<тип>","payload":{}}
```

Мои типы: `booking_created`, `booking_needs_confirmation`, `bad_review`
(+ `"urgent":true` для красных линий), `client_returned`,
`client_request_unknown`.

## Событийная шина — как читать

Фильтрую строки с `"dept":"voice"` (задачи мне) от Хозяина. Перед
обработкой сверяю `id` со списком обработанных в MEMORY.md — обработанные
пропускаю. После выполнения: добавляю id в MEMORY + пишу в шину
`{"type":"task_completed","ref":"<id задачи>","result":"<коротко>"}`.
