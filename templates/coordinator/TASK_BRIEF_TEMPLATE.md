# TASK_BRIEF_TEMPLATE.md — Шаблон задания агенту

> Координатор заполняет этот шаблон ПЕРЕД каждым `sessions_send`.
> Без заполненного брифа задание не уходит.

---

## Шаблон (копировать и заполнять)

```
TASK_BRIEF
==========
Агент: [copywriter / marketer / designer / tech / producer]
Дата: [ДД.ММ.ГГГГ ЧЧ:ММ <<часовой пояс>>]
Приоритет: [Высокий / Средний / Низкий]

ЦЕЛЬ:
[Одна строка — что конкретно должно получиться на выходе]

ЧТО СДАТЬ:
- Файл: [путь, например ~/.openclaw/workspace-<агент>/outputs/имя_файла.md]
- Формат: [.md / .html / .png / другой]
- Дополнительно: [отправить файлом в чат владельца <<telegram-id>>]

ВХОДЫ (что использовать):
- [файл 1: путь]
- [файл 2: путь]
- [ссылка или контекст]

ЧЕГО НЕ ДЕЛАТЬ (границы задачи):
- [ограничение 1]
- [ограничение 2]
- [не выходить за роль — пример: "не писать тексты, только структуру"]

ПРОВЕРИТЬ ПЕРЕД СДАЧЕЙ (чек-лист):
- [ ] Открыть PRE_SEND_CHECKLIST.md и пройти его
- [ ] Файл реально сохранён по указанному пути
- [ ] Записал в MEMORY.md: "Сдано: [имя_файла], [дата]"
- [ ] Файл отправлен в чат владельца с подписью

КОНТЕКСТ:
[Ссылки на MEMORY.md / другие файлы / пояснение зачем это нужно]
```

---

## Правило MAX_ITERATIONS=8

Если агент 8 раз пробовал выполнить задачу и не завершил —
остановиться и написать в чат владельца:
`🚨 Застрял после 8 попыток — нужен <<имя владельца>>. Задача: [название]. Где застрял: [описание].`

Не уходить в бесконечный цикл переформулировок.

## Пример заполненного брифа

```
TASK_BRIEF
==========
Агент: copywriter
Дата: <<ДД.ММ.ГГГГ ЧЧ:ММ>>
Приоритет: Высокий

ЦЕЛЬ:
<<написать тексты блоков hero + проблема + решение для главного лендинга>>

ЧТО СДАТЬ:
- Файл: ~/.openclaw/workspace-copywriter/outputs/<<имя_файла_с_датой>>.md
- Формат: .md
- Дополнительно: отправить файлом в чат владельца с короткой подписью

ВХОДЫ (что использовать):
- <<путь к скелету / брифу>>
- MEMORY.md (раздел <<какой именно>>)
- <<регалия / позиционирование, как пишется в публичных текстах>>

ЧЕГО НЕ ДЕЛАТЬ:
- <<не использовать запрещённые слова из MEMORY.md «Жёсткие правила»>>
- <<не добавлять обещаний, которые не зафиксированы как факт>>
- <<не раскрывать цену в местах, где она не предусмотрена>>
- <<не писать в стиле жёсткой продажи / не уходить в канцелярит>>

ПРОВЕРИТЬ ПЕРЕД СДАЧЕЙ:
- [ ] Открыть PRE_SEND_CHECKLIST.md и пройти его
- [ ] Файл реально сохранён
- [ ] Записал в MEMORY.md что сдано
- [ ] Файл отправлен в чат владельца

КОНТЕКСТ:
<<2-3 строки: зачем это сейчас, что было до, что будет после>>
```
