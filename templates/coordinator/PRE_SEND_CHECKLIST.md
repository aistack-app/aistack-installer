# PRE_SEND_CHECKLIST.md — Перед отправкой результата

> Каждый агент перед сдачей задачи проходит этот чек-лист.
> Координатор отказывает в приёмке если что-то не отмечено.

---

## Чек-лист

- [ ] **Файл существует физически** по указанному в брифе пути
- [ ] **Формат соответствует заданию** (.md / .html / .png / .pdf и т.д.)
- [ ] **Имя файла читаемое** — `что_когда_версия.расширение`
- [ ] **В MEMORY.md моего workspace** записал «Сдано: <имя>, <дата>»
- [ ] **Прошёл по ограничениям из брифа** (раздел «ЧЕГО НЕ ДЕЛАТЬ»)
- [ ] **Проверил по SYSTEM_STATUS координатора** — нет ли свежих правил
       которые меняют результат (например «не использовать слово X»)
- [ ] **В подписи к файлу** в чате — коротко: что внутри + что от пользователя надо

---

## Если чек-лист не прошёл

- НЕ отправлять
- Дописать недостающее
- Если непонятно — спросить координатора через `handoff/`

---

## Антипаттерны при отправке

❌ Отправить файл без подписи
❌ Подпись «вот, готово» без описания что внутри
❌ Подпись «вот, готово, посмотри» — пользователь не понимает что делать
❌ Файл лежит в неправильной папке (`Desktop` вместо `outputs`)
