# PROJECT_HANDOFF_SCHEMA.md

## Зачем

Этот файл нужен, чтобы ты мог передавать контекст по проектам без хаоса,
а Координатор потом:
- не жёг лишние токены на повторное чтение всего подряд
- не смешивал факты и гипотезы
- мог разносить контекст по профильным агентам

## Главный принцип

Лучше **один сырой файл + короткая подпись**, чем 20 сообщений вразнобой.

Я сначала складываю сырой контекст в проектную папку, потом делаю:
1. master-brief
2. agent-handoffs
3. обновление `USER.md` / `MEMORY.md` у нужных агентов

---

## Как передавать материалы

### Вариант A — лучший

Присылай **файл** и короткую подпись по шаблону:

```text
Проект: [проект1] или [проект2]
Тип: факты / черновик / гипотезы / ТЗ / переписка / файл от Claude
Что с этим сделать: сохранить / разобрать / сжать / разнести по агентам / проверить на противоречия
Насколько это актуально: высоко / средне / потом
```

Пример:

```text
Проект: [проект1]
Тип: файл от Claude
Что с этим сделать: сохранить и разнести по агентам
Насколько это актуально: высоко
```

### Вариант B — если текста мало

Можно просто одним сообщением:

```text
Проект: [проект2]
Тип: гипотеза
Текст: ...
```

---

## Что лучше присылать отдельными файлами

Отдельным файлом лучше присылать:
- большие ответы из Claude
- продуктовые описания
- таблицы / структуры воронки
- наброски лендинга
- технические схемы
- юридические правила / policy / safety-блоки
- большие переписки

Не надо вставлять это всё в чат кусками, если можно отправить файлом.

---

## Как я это буду раскладывать

Для каждого проекта держим 4 слоя:

### 1. Raw
Сырые материалы без сильной переработки.

### 2. Facts
Что подтверждено точно.

### 3. Hypotheses
Что пока только идея / версия / направление.

### 4. Agent handoff
Короткий профильный контекст по ролям:
- marketer
- producer
- tech
- designer
- copywriter

---

## Что важно помечать

Если материал из Claude, обязательно помечать одним из вариантов:
- `факт`
- `черновик`
- `гипотеза`
- `надо проверить`

Потому что Claude может хорошо писать, но это не значит, что всё там правда
или уже решено.

---

## Минимальный формат для любого нового файла

В начале сообщения или подписи к файлу достаточно 4 строк:

```text
Проект: [проект1]/[проект2]
Источник: Claude / я / <<имя соавтора>> / агент / сторонний файл
Статус: факт / черновик / гипотеза / проверить
Действие: сохранить / сжать / разнести / превратить в ТЗ
```

---

## Какой рабочий режим самый выгодный по токенам

Самый дешёвый путь такой:

1. Ты кидаешь **сырой файл**
2. Я делаю **короткую выжимку**
3. Я отдельно делаю **handoff по ролям**
4. Только потом это уходит профильным агентам

То есть:
**не все агенты читают весь Claude-файл**, а каждый получает только своё.

Это и дешевле, и чище.

2. отделяю факты от гипотез
3. делаю master-brief
4. обновляю память
5. готовлю задания агентам
