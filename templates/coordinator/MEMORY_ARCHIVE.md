# MEMORY_ARCHIVE.md — Архив решений

> Исторический лог решений и важных событий. Активная рабочая память — в `MEMORY.md`.
> Сюда переносится всё, что старше 2-3 недель или потеряло актуальность для текущей работы, но может пригодиться как контекст потом.

## Зачем этот файл

- Не превращать активную `MEMORY.md` в свалку
- Иметь возможность вспомнить «почему мы тогда решили так»
- При работе с агентами через несколько месяцев — поднимать контекст принятых решений

## Структура

Хронологически в обратном порядке (свежие сверху). Каждое решение — короткое, 1-3 строки.

## Хронология ключевых решений

### <<ГГГГ-ММ-ДД>> (свежие сверху)
- <<решение / событие>>
- <<кто что зафиксировал>>

### <<ГГГГ-ММ-ДД>>
- <<решение>>
- <<контекст в одной строке>>

### <<ГГГГ-ММ-ДД>>
- <<решение>>

## Отменённые / устаревшие решения

> Сюда переносить решения, которые **больше не действуют**. Полезно, чтобы не повторять отказанные варианты.

- ~~<<отменённое решение>>~~ — отменено <<дата>> по причине <<причина>>

## Примечание

- Не вставлять сюда переписку. Только итоги: что решили, дата, краткая причина.
- При каждом обновлении `MEMORY.md` посмотреть — есть ли там старые блоки которые пора в архив.
