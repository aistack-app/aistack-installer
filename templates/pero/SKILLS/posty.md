# Навык: Посты в ленту

## Структура поста

1. **Крюк** (1 строка) — боль, интрига, цифра или сцена. Не приветствие.
2. **Тело** (3–7 строк) — одна мысль, конкретика, живые детали.
3. **CTA** (1 строка) — запись / вопрос / «сохрани».

Длина по умолчанию — до 600 знаков. Лонгрид — только по запросу.

## Типы постов (чередовать, не зацикливаться на продаже)

- **Продающий** — услуга + выгода + слот + CTA на запись.
  Не чаще 1 из 3 постов.
- **Экспертный** — разрушить миф, ответить на частый вопрос клиентов
  (вопросы беру из памяти Голоса через владельца/Хозяина).
- **Закулисье** — мастер, процесс, материалы, «как мы выбираем X».
- **Социальное доказательство** — отзыв/результат (с разрешения
  клиента! — напоминаю владельцу спросить).
- **Анонс/акция** — факты из BUSINESS.md, дедлайн, простое условие.

## Чек-лист перед выдачей

- [ ] Факты сверены с BUSINESS.md (цены, имена, график)
- [ ] Первая строка работает без остального текста
- [ ] Одна мысль, один CTA
- [ ] Тон совпадает с «Тон бренда» из BUSINESS.md
- [ ] Нет канцелярита, нет AI-штампов
- [ ] Если упомянут клиент/фото клиента — пометка владельцу «нужно
      разрешение»

## После публикации

Спрашиваю у владельца отклик через день-два (или беру из его слов) →
фиксирую в MEMORY.md: тема, тип, зашло/нет. Через 10+ постов память
сама покажет, что работает в этой студии.
