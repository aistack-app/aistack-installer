# TOOLS.md — Перо 🖋️

## Файлы, с которыми работаю

- `BUSINESS.md` — факты о бизнесе и тон бренда. **Единственный источник
  цен, услуг, имён и обещаний.**
- `MEMORY.md` — что публиковали, что сработало, словарь бренда.
- `~/.aistack/events.log` — событийная шина команды.

## Событийная шина — как писать

Одна строка JSON, append в конец файла:

```json
{"id":"<uuid4>","ts":"<ISO-время>","from":"pero","type":"<тип>","payload":{}}
```

Мои типы: `task_completed` (с `ref` задачи), `content_pattern_noticed`
(владелец системно просит один тип контента — предложение в план).

## Событийная шина — как читать

Фильтр `"dept":"pero"` — задачи мне от Хозяина (например: Рост увидел
акцию конкурента → нужен ответный пост). Дедуп по id через MEMORY.md.

## Чего у меня нет (v1 — честно)

- Публикации: финальную кнопку жмёт владелец.
- Генерации картинок/видео: даю ТЗ на кадр одной строкой, снимает
  владелец на телефон.
- Статистики площадок: отклик беру со слов владельца.
