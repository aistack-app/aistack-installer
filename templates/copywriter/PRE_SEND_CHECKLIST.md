## Обязательные проверки

### 🚫 Запрещённые слова — грепать в артефакте

> Список запрещённых формулировок берётся из `MEMORY.md` раздела «Жёсткие правила» (а также `USER.md` «Запреты / стоп-формулировки»).
> Команду собрать один раз под свой проект и держать здесь.

```bash
grep -i "<<запрещённое слово 1>>\|<<запрещённое слово 2>>\|<<запрещённое слово 3>>" <файл>
```

- [ ] Не найдено `<<запрещённое слово 1>>`
- [ ] Не найдено `<<запрещённое слово 2>>`
- [ ] Не найдено паразитных конструкций (`не просто X, а Y`, длинных тире без смысла, канцелярита: «качественный», «эффективный», «значительный»)

### ✅ Обязательные проверки содержания

- [ ] Регалия эксперта: **только утверждённая формулировка** из `USER.md`
- [ ] Имя на формальных блоках / лендингах — полная форма (`<<Имя Фамилия>>`), не уменьшительная
- [ ] Контакты верные: <<TG handle>>, <<email>>, <<домен>> (взято из `USER.md`)
- [ ] Цена упоминается **только там, где разрешено** (см. `USER.md` — правила по сайтам)
- [ ] Ключевые даты (эфир / старт потока / окно продаж) совпадают с `USER.md` / `MEMORY.md`
- [ ] Юридический статус и формулировки соответствуют согласованным
- [ ] Голос клиента из `MEMORY.md` «Словарь клиента» использован — не «голос эксперта», не моя экспертная лексика

### 📁 Проверка файла

- [ ] Файл реально сохранён по пути из брифа
- [ ] Размер файла > 0 байт
- [ ] Кодировка UTF-8, читается нормально

### 📝 Фиксация в памяти

- [ ] Записал в `MEMORY.md` строку: `Сдано: [имя_файла], [дата], статус: на апруве`

### 📤 Доставка

- [ ] Файл отправлен в чат владельца
- [ ] Подпись к файлу написана (одна строка: что это и откуда)

## Если нашёл запрещённое слово

1. Исправить во всём файле (не только в найденном месте — искать все вхождения)
2. Прогнать grep снова — убедиться что чисто
3. Только потом отправлять

## Правило MAX_ITERATIONS=8

Если 8 раз пробовал и задача не закончена — **стоп**.
Пишу владельцу: `🚨 Застрял после 8 попыток. Задача: [название]. Где: [проблема].`
